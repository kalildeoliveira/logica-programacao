Aula 1 - Nomes e tipos de Variáveis

Se liga nestes tópicos:

  • Conceito: Se você lembrar das aulas de matemática lá no Ensino Fundamental, com certeza não vai ter dificuldade em entender o que são variáveis. Elas eram o "x" ou "y" nas suas equações. Fácil né? Elas servem como caixinhas ou espaços na memória para guardar dados.
  • Algns exemplos importantes são:
    • String ou cadeia de caracteres, uteis para guardar o nome de uma pessoa ou para números cujo propósito não será fazer cálculos, como o CPF de alguém, por exemplo;
    • Inteiro, ideal para fazer contagem ou operações que não retornam fração ou números com vírgula;
    • Float ou Real para números com vírgula ou frações ou seja, que possa haver metade de um número, por exemplo ;
    • Boleana ou Lógica para guardar valor verdadeiro ou falso e ajudar nas decisões;
    • Array, vetores e matrizes que são estruturas para guardar vários dados, como uma tabela;
    • Objetos que imitam o mundo real e, além de valores ou atributos, são capazes de guardar comportamentos ou métodos.
  • Quando vamos criar ou declarar uma variável é preciso cuidar para não deixar espaço nem usar caracteres especiais, acentuação ou iniciar com números. Os programas aceitam variáveis com $ e _, mas o mais correto é iniciar com minúsculo e, caso precisar de uma segunda palavra, usar uma maiúscula. Assim: nomeUsuario.
  • Uma sugestão é criar variáveis que sejam intuitivas, ou seja, numeroId é um bom nome para guardar um número de identificação e não "n". É claro que funciona e na hora você até vai lembrar, mas e depois? Lembre-se que os códigos não são eternos e precisam de manutenção no futuro.
  • Enfim, uma variável misteriosa vai dificultar que as pessoas entendam o seu código ou você mesmo quando precisar melhorá-lo.
